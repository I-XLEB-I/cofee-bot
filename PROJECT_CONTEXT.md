## Бизнес-логика обслуживания

### Режим обслуживания
Точки обслуживаются "через день":
- если точка была обслужена вчера, сегодня её можно не обслуживать
- если была позавчера или раньше, она попадает в список "нужно сегодня"

### Блок `К обслуживанию сегодня`
Есть отдельные блоки:
- `🚨 Срочно пополнить`
- `🔴 Нужно сегодня`
- `🟡 На контроле`
- `⏭ Можно не обслуживать сегодня`
- `✅ Уже обслужены сегодня`

### Критический сигнал
Точка попадает в критический блок, если по расходнику указано:
- `запаса для пополнения нет`
- и `не хватит до следующего приезда`

## Даты

### В обслуживании и исправлении записей
Ручной ввод даты:
- формат: `дд.мм`
- пример: `05.04`

Дополнительно:
- зимние даты вроде `30.12` автоматически понимаются как прошлый год
- дату нельзя указывать позже сегодняшней
- дату нельзя указывать старше 1 года

### Важно
Бот работает по московскому времени.

## Нехватка расходников

Логика нехватки сейчас описывает не весь остаток на точке, а именно:
`сколько осталось в запасе для пополнения`

Для выбранного расходника можно:
- указать остаток в запасе
- нажать `Пропустить`
- нажать `Запаса для пополнения нет`

Если выбран `Запаса для пополнения нет`, бот дополнительно спрашивает:
- `Хватит до следующего приезда`
- `Не хватит до следующего приезда`

Это влияет на критичность точки в блоке `К обслуживанию сегодня`.

### Единицы в нехватке
- Стаканы: туб
- Крышки бел: туб
- Крышки чёрн: туб
- Сахар: стиков
- Манжеты: пачек
- Палочки: пачек
- Трубочки: пачек
- Сиропы: бутылок
- Остальное по текущим единицам в коде

### Отображение
Нехватка должна показываться списком, а не одной длинной строкой.

Пример правильного вида:
- Стаканы — запас: 1 туб
- Сахар — запаса для пополнения нет, не хватит до следующего приезда

## Ревизия

Ревизия — это отдельный блок, не смешанный с обычным обслуживанием.

### Период ревизии
Ревизия хранится по месяцам:
- Март 2026
- Апрель 2026
и т.д.

То есть выбирается месяц, а не конкретный день.

### Локации ревизии
- все точки
- Дома

### Порядок товаров в ревизии
Строго в таком порядке:
1. Кофе
2. Молоко
3. Мока
4. Шоколад
5. Стаканы
6. Сахар
7. Сиропы
8. Крышки бел
9. Крышки чёрн
10. Трубочки
11. Палочки
12. Манжеты
13. Вода
14. Влажные салф
15. Мус.пакеты

### На каждом шаге ревизии
Должны быть:
- быстрые кнопки
- `Другое`
- `Пропустить`
- `Назад`

### Что уже есть
- Заполнить ревизию
- Изменить ревизию

### Изменение ревизии
Нужны режимы:

### Просмотр ревизии
Есть режимы:
- общая по месяцу
- по точке

### Сравнение
Сравнение делается по точке:
- текущий месяц vs прошлый месяц

## Импорт ревизии из текста

Есть отдельная функция:
`📥 Импорт из текста`

Она нужна для старых ревизий, которые сотрудники прислали в чат вручную, когда бот ещё не поддерживал ревизию или не было интернета на точке.

### Какие форматы понимает импорт
1. Блоки по точкам:
```text
02.04 Бел2
Кофе - 3.1
Молоко - 1.4
...
```

2. Табличный формат через `/`:
```text
Сити/юж/бел/б2/гагар/гиппо/дома
Кофе исп - 4/0/5/0/5/5/
Кофе кит - 0/5/0/3/0/0/
...
```

### Особенности импорта
- `Кофе исп` и `Кофе кит` складываются в общее поле `Кофе`
- добавлена поддержка `Салфетки сухие`, потому что они встречаются в исторической ревизии
- перед сохранением бот показывает превью импорта
- бот показывает предупреждения о конфликтующих значениях
- для обычных товаров при конфликте сейчас берётся последнее найденное значение
- для кофе исп/кит значения суммируются

### Практический смысл
Это позволяет не переносить ревизию вручную из старых сообщений.

## UX-договорённости

- `Назад` должен возвращать на предыдущий логический шаг
- старые сообщения в Telegram не обновляются автоматически после изменений кода
- после обновления бота экран нужно открывать заново
- длинные технические строки лучше не показывать пользователю, если можно дать компактную карточку
- подтверждения и карточки должны быть короткими и быстро читаемыми
